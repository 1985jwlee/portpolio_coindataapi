# PrevPortfolio WebServer  
실시간 외부 데이터 수집을 위한 이벤트 기반 웹 서버 아키텍처

---

## 1. 프로젝트 개요

PrevPortfolio WebServer는 **외부 데이터 소스로부터 실시간 시계열 데이터를 수집하고,  
이를 서버 프로세스 형태로 제공하기 위해 설계·구현한 웹 서버 프로젝트**이다.

이 프로젝트는 단순한 API 서버나 기술 데모가 아니라,

- 외부 데이터 의존성이 높은 환경에서
- 지속 실행되는 서버 프로세스를 안정적으로 운영하고
- 데이터 흐름을 이벤트 중심으로 제어하는

**실전형 백엔드 시스템 설계 경험을 정리한 포트폴리오**다.

---

## 2. 설계 목표 (Why)

이 프로젝트를 통해 해결하고자 한 핵심 문제는 다음과 같다.

1. 외부 데이터 소스는 **언제든 불안정해질 수 있다**
2. 실시간 데이터는 **동시성과 순서 보장이 중요하다**
3. 서버는 코드 묶음이 아니라 **운영되는 프로세스**여야 한다
4. 데이터 소스가 늘어나도 구조는 흔들리지 않아야 한다

따라서 본 프로젝트의 설계 목표는 다음과 같이 정의했다.

- 이벤트 기반 데이터 흐름 설계
- 인터페이스 중심 구조로 확장성 확보
- 동시성 문제를 고려한 데이터 관리
- 운영을 전제로 한 서버 프로세스 구성

---

## 3. 기술 스택

- **C# (.NET Core 기반)**
- 비동기 / 이벤트 기반 처리 모델
- 인터페이스 중심 아키텍처
- 멀티 플랫폼 빌드 (Windows / Linux)

---

## 4. 전체 아키텍처 개요

Program (Entry Point)
├─ WebServer
│ ├─ Request Handler
│ └─ Data Response Layer
├─ Data Collector
│ ├─ External Source Adapter
│ └─ Event Dispatcher
├─ Interfaces
│ ├─ IDataProvider
│ └─ IProcess
├─ Domain / Common
└─ Concurrent Data Store


각 계층은 **명확한 역할 분리**를 기준으로 구성되며,  
서로는 인터페이스를 통해서만 의존하도록 설계되었다.

---

## 5. 핵심 설계 판단

### 5.1 인터페이스 중심 구조

외부 데이터 소스는 언제든 교체·추가될 수 있기 때문에  
구현체가 아닌 **계약(Interface)** 에 의존하도록 설계했다.

- 데이터 수집 로직과 서버 로직 완전 분리
- 신규 데이터 소스 추가 시 기존 코드 수정 최소화
- 테스트 및 대체 구현 용이

---

### 5.2 이벤트 기반 데이터 흐름

데이터 수집 → 처리 → 제공 흐름을  
**동기 호출이 아닌 이벤트 흐름으로 모델링**했다.

이를 통해:

- 데이터 유입 속도 변화에 유연하게 대응
- 처리 병목 구간 분리
- 실시간 시스템에서 발생하는 지연 전파 최소화

---

### 5.3 동시성 및 안정성 고려

실시간 데이터 특성상:

- 다수의 요청이 동일 데이터에 접근
- 수집 중 데이터 변경 가능성

이 문제를 해결하기 위해:

- 동시 접근이 가능한 자료구조 사용
- 읽기/쓰기 분리 개념 적용
- 데이터 불일치 가능성 최소화

---

### 5.4 “서버는 프로세스다”라는 관점

이 프로젝트는 라이브러리가 아닌 **운영 가능한 서버 프로세스**를 목표로 했다.

- 실행 가능한 바이너리 제공
- 설정값 외부화(JSON)
- 장시간 실행을 전제로 한 구조

이는 “돌아가는 코드”가 아니라  
**운영되는 시스템**을 만든다는 관점의 결과다.

---

## 6. 운영 및 장애 상황 가정

본 프로젝트는 아래와 같은 현실적 상황을 가정했다.

- 외부 데이터 소스 연결 단절
- 데이터 수신 지연 또는 불규칙한 유입
- 서버 재시작 상황

이에 따라:

- 데이터 수집과 제공 로직 분리
- 서버 재기동 시 빠른 정상화 가능 구조
- 상태를 최소화한 서버 설계

---

## 7. 의도적으로 하지 않은 것들

- 도메인 특화 로직(트레이딩, 분석 등) ❌
- 복잡한 비즈니스 규칙 ❌
- UI 또는 클라이언트 구현 ❌

본 프로젝트는  
**데이터 파이프라인과 서버 구조 자체에 집중**하기 위해 범위를 제한했다.

---

## 8. 이 포트폴리오의 위치

- 과거 프로젝트이지만  
- 현재의 커리어 방향(실시간·이벤트 기반 시스템 설계)을  
  가장 잘 설명해주는 **기초 사례**

이 프로젝트는 이후의:

- 실시간 게임 서버
- 이벤트 스트리밍 기반 플랫폼
- 메시지 브로커 중심 아키텍처

로 이어지는 사고 방식의 출발점에 해당한다.

---

> 설계 관점에서 구현한 포트폴리오**다.
